PROJECT GOAL

- Exposes a POST /events endpoint that Zapier can call via Webhooks.

- Logs all incoming webhook payloads and keeps a simple in-memory history so I can see what Zapier is sending.
- Is easy to run locally with uvicorn and expose via ngrok.

- Use uvicorn as the ASGI server.

ENDPOINT DETAILS

1) POST /events
- Purpose: receive webhook calls from Zapier.
- Expect JSON body with at least:
  - event_id: str
  - source: str
  - payload: dict | Any
- If event_id is missing, generate a UUID on the server side.
- Validate the incoming data using Pydantic models.
- Store each received event in a simple in-memory store (a list in storage.py is fine for now, with a max size like 100 events and FIFO eviction).
- Log each incoming event with:
  - Timestamp
  - event_id
  - source
  - size of payload (len of JSON string or number of keys)
- Return JSON:
  {
    "status": "ok",
    "event_id": "<resolved_id>",
    "stored_at": "<iso8601 timestamp>"
  }

2) GET /status
- Purpose: quick health and debug endpoint.
- Return:
  - service: "zapier-webhook-receiver"
  - status: "healthy"
  - events_received: total number of events stored in memory
  - last_event: a small summary of the most recent event if any (event_id, source, received_at), or null if none.
- Use a Pydantic response model.
